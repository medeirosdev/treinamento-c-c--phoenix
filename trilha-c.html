Fundamentos do C

Em progresso

Por que C?

C é a linguagem que moldou a computação moderna. O kernel do Linux, o interpretador do Python, o Arduino — tudo escrito em C. É rápida, próxima do hardware e oferece controle total de memória. Entender C é entender como o computador realmente funciona.

Estrutura mínima de um programa

Todo programa C começa com a função main(). O sistema operacional chama main quando o programa é executado.

#include <stdio.h>   // biblioteca de I/O padrão

int main(void) {
    printf("Olá, mundo!\n");  // \n = nova linha
    return 0;              // 0 = sucesso pro sistema operacional
}

Processo de compilação

Seu arquivo .c não vai direto para execução. Passa por 4 etapas:

LinkerJunta .o + bibliotecas.o → executável

./meu_programa

Sufixos de literais numéricos

C infere o tipo de constantes numéricas. Sufixos explicitam o tipo e evitam conversões silenciosas:

SufixoTipo resultanteExemplo
f ou Ffloat (32 bits)3.14f
(nenhum)double (64 bits)3.14
Llong ou long double1000L
u ou Uunsigned int300u
ul ou ULunsigned long999999ul
float a  = 3.14f;     // sem f: literal double → conversão implícita para float
long  c  = 100000L;  // sem L: literal int, pode não caber em int de 16 bits
unsigned d = 50u;   // unsigned deixa claro que não é negativo
Em robótica com MCU de 32 bits (ESP32, STM32), use 3.14f e não 3.14 em operações float. Sem o f, o compilador opera em double (64 bits) e pode ser mais lento em hardware sem FPU de 64 bits.

Escopo: global vs local

O escopo define onde uma variável é visível e quando ela existe na memória:

int vel_max = 255;      // GLOBAL: visível em todo o arquivo, durante toda execução

void acelerar(int alvo) {
    int incremento = 10; // LOCAL: existe só nesta função, destruída ao retornar
    if (alvo > vel_max) alvo = vel_max;  // acessa global
}   // incremento deixa de existir aqui

void frear(void) {
    vel_max = 0;        // mesma global, outra função
    // incremento não existe aqui — escopo local é exclusivo
}
Evite variáveis globais: Elas criam dependências ocultas entre funções, dificultam testes e escondem bugs. Passe dados como parâmetros. Em robótica, globais são às vezes necessárias (ex: buffer de leitura de sensor em interrupção), mas use com consciência.

Operadores aritméticos

int a = 10, b = 3;
printf("%d\n", a + b);   // 13
printf("%d\n", a - b);   // 7
printf("%d\n", a * b);   // 30
printf("%d\n", a / b);   // 3  ← divisão inteira! trunca
printf("%d\n", a % b);   // 1  ← resto (módulo)
Armadilha de divisão inteira: 10 / 3 em C resulta em 3, não 3.333... Para obter float, converta: (float)a / b ou use 10.0 / 3.

Operadores de comparação e lógicos

int x = 5;
x == 5   // igual?      → 1 (verdadeiro)
x != 3   // diferente?  → 1
x >  3   // maior?      → 1
x <= 5  // menor igual → 1

// Lógicos
x > 0 && x < 10   // AND: ambos verdadeiros?
x < 0 || x > 100  // OR: pelo menos um?
!0               // NOT: inverte → 1
= vs ==: if (x = 5) atribui 5 a x (sempre verdadeiro!). if (x == 5) compara. Esse bug é silencioso e perigoso. Muitos programadores escrevem if (5 == x) para forçar erro de compilação se usarem = por acidente.

Operadores bitwise

Extremamente importantes em sistemas embarcados para manipular registradores e flags:

uint8_t reg = 0b00101010;  // valor binário

reg |  0b00000001   // OR:  seta bit 0
reg & ~0b00000001   // AND NOT: limpa bit 0
reg ^  0b00000100   // XOR: inverte bit 2
reg << 1            // shift left:  multiplica por 2
reg >> 1            // shift right: divide por 2

// Exemplo prático: setar bit 3 sem afetar outros
reg |= (1 << 3);  // equivale a reg = reg | 0b00001000

Incremento e operadores compostos

int n = 5;
n++;      // n = 6  (pós-incremento: usa n depois incrementa)
++n;      // n = 7  (pré-incremento: incrementa antes de usar)
n--;      // n = 6
n += 3;   // n = 9   (equivale a n = n + 3)
n *= 2;   // n = 18
n /= 6;   // n = 3
n %= 2;   // n = 1

Cast de tipo

int grande = 300;

switch / case

Use quando comparar uma variável inteira contra valores fixos:

int dia = 3;
switch (dia) {
    case 1: printf("Segunda\n"); break;
    case 2: printf("Terça\n");   break;
    case 3: printf("Quarta\n");  break;
    // sem break → "fall-through": executa case seguinte também!
    case 6:
    case 7: printf("Fim de semana\n"); break;
    default: printf("Inválido\n");
}
Fall-through: Esquecer o break faz o código "cair" no próximo case. Às vezes intencional (case 6 e 7 acima), mas quase sempre um bug.

Loop while

int i = 0;
while (i < 5) {
    printf("%d ", i);  // 0 1 2 3 4
    i++;
}

Loop for

// for (inicialização; condição; passo)
for (int i = 0; i < 5; i++) {
    printf("%d ", i);
}

// Iterando de trás pra frente
for (int i = 4; i >= 0; i--) {
    printf("%d ", i);  // 4 3 2 1 0
}

// Loop infinito controlado
for (;;) {
    // executa forever, sai com break
    break;
}

Loop do-while

Executa o corpo pelo menos uma vez, depois verifica a condição:

int n;
do {
    printf("Digite um número positivo: ");
    scanf("%d", &n);
} while (n <= 0);  // repete até ser positivo

break e continue

for (int i = 0; i < 10; i++) {
    if (i == 3) continue;  // pula i=3, vai pro próximo
    if (i == 7) break;     // sai do loop ao chegar em 7
    printf("%d ", i);      // 0 1 2 4 5 6
}

Protótipos (declaração antecipada)

O compilador lê o arquivo de cima pra baixo. Se você chamar uma função antes de defini-la, ele reclama. Protótipos resolvem isso:

// Protótipo: informa a assinatura antes da definição
float media(float a, float b);  // ← ponto e vírgula!

int main(void) {
    printf("%.1f\n", media(8.0f, 6.0f));  // 7.0
    return 0;
}

float media(float a, float b) {
    return (a + b) / 2.0f;
}

Passagem por valor vs por referência

void tentaTrocar(int a, int b) {
    int tmp = a; a = b; b = tmp;
    // a e b são CÓPIAS — original não muda!
}

void trocar(int *a, int *b) {  // ponteiros: "passa o endereço"
    int tmp = *a; *a = *b; *b = tmp;
    // *a e *b acessam os originais — funciona!
}

int x = 1, y = 2;
trocar(&x, &y);   // x=2, y=1

Escopo de variáveis

int global = 10;  // visível em todo o arquivo

void funcao(void) {
    int local = 5;    // existe só dentro da função
    static int conta = 0;  // persiste entre chamadas!
    conta++;
    printf("chamada %d\n", conta);
}
static em variável local: o valor é mantido entre chamadas. Útil para contadores e acumuladores dentro de funções (muito usado em PID, onde precisamos do erro anterior).

Recursão

int fatorial(int n) {
    if (n <= 1) return 1;  // caso base: para a recursão
    return n * fatorial(n - 1);  // chamada recursiva
}
// fatorial(4) = 4 * fatorial(3) = 4 * 3 * 2 * 1 = 24
Recursão sem caso base correto → loop infinito e stack overflow. Em sistemas embarcados, evite recursão profunda pois a pilha é limitada (geralmente poucos KB).

Como pensar em recursão: caminho de ida e volta

Toda recursão tem dois momentos — André Backes chama de caminho de ida e caminho de volta:

  • Caminho de ida: a função chama a si mesma, aproximando-se do caso base. Cada chamada empilha um novo frame na pilha.
  • Caso base: a condição de parada. Sem ela, recursão infinita e stack overflow.
  • Caminho de volta: o caso base retorna, e as chamadas anteriores completam seu cálculo em ordem inversa.
// fatorial(4): rastreando cada frame
int fatorial(int n) {
    if (n <= 1) return 1;     // caso base: para a descida
    return n * fatorial(n-1);  // chama com n reduzido (critério de parada)
}

// IDA:    fatorial(4) → fatorial(3) → fatorial(2) → fatorial(1)
// BASE:   fatorial(1) retorna 1
// VOLTA:  fatorial(2) retorna 2*1=2
//         fatorial(3) retorna 3*2=6
//         fatorial(4) retorna 4*6=24  ← resultado final
Duas regras para recursão funcionar: (1) o critério de parada (caso base) deve ser atingível; (2) o parâmetro da chamada recursiva deve se aproximar do caso base a cada chamada. Se n nunca chegar a 1, a recursão nunca para.

Recursão vs Iteração: Fibonacci

Fibonacci é o exemplo clássico de onde recursão é elegante, mas ineficiente:

// Versão RECURSIVA — elegante, porém lenta: O(2^n)
int fib_recursivo(int n) {
    if (n == 0) return 0;
    if (n == 1) return 1;
    return fib_recursivo(n-1) + fib_recursivo(n-2);
}
// fib(10) faz ~177 chamadas. fib(40) faz ~331 milhões de chamadas!

// Versão ITERATIVA — eficiente: O(n)
int fib_iterativo(int n) {
    if (n <= 1) return n;
    int a = 0, b = 1, c;
    for (int i = 2; i <= n; i++) {
        c = a + b;   // próximo Fibonacci
        a = b;       // avança a janela
        b = c;
    }
    return b;
}
Fibonacci recursivo recalcula o mesmo valor muitas vezes. fib(5) chama fib(3) duas vezes, fib(2) três vezes. Para n grande, o tempo explode exponencialmente. Use a versão iterativa ou técnicas como memoização.

Passando arrays para funções

Quando um array é passado para uma função, ele "decai" (decay) para um ponteiro para o primeiro elemento. O tamanho é perdido — passe sempre como parâmetro separado:

// As duas declarações abaixo são equivalentes:
float media(float *arr, int n);       // notação de ponteiro
float media(float  arr[], int n);    // notação de array — mesmo que acima!

float media(float arr[], int n) {
    float soma = 0;
    for (int i = 0; i < n; i++) soma += arr[i];
    return soma / n;
}

float sensores[9] = {0,0,1,1,1,0,0,0,0};
float m = media(sensores, 9);  // passa ponteiro + tamanho explícito
Dentro de media(), você não consegue saber o tamanho do array com sizeof(arr) — ele retorna o tamanho do ponteiro (4 ou 8 bytes), não do array. Sempre passe o tamanho como parâmetro separado.

Ponteiros

Em progresso

O que é um ponteiro?

Uma variável que armazena um endereço de memória. Em vez de guardar o dado, guarda onde o dado está.

int x = 42;
int *p = &x;    // p aponta para x (&x = "endereço de x")

printf("%d\n", x);    // 42    — o valor
printf("%p\n", p);    // 0x... — o endereço
printf("%d\n", *p);   // 42    — desreferência: valor no endereço

*p = 100;            // modifica x através do ponteiro
printf("%d\n", x);   // 100

Ponteiro nulo

int *p = NULL;    // ponteiro que não aponta pra nada

if (p != NULL) {
    *p = 5;       // seguro: verifica antes de usar
}
// *NULL → Segmentation Fault imediato (crash)

Aritmética de ponteiros

int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;    // arr decai para ponteiro pro primeiro elemento

printf("%d\n", *p);       // 10
printf("%d\n", *(p+1));   // 20 (avança sizeof(int) bytes)
printf("%d\n", p[2]);     // 30 (notação de array = aritmética de ponteiros)

p++;  // p agora aponta para arr[1]
arr[i] é exatamente igual a *(arr + i). A notação de array é apenas açúcar sintático sobre aritmética de ponteiros.

Ponteiro para ponteiro

int   x  = 5;
int  *p  = &x;   // ponteiro para int
int **pp = &p;   // ponteiro para ponteiro para int

printf("%d\n", **pp);  // 5 — dois níveis de desreferência

Dangling pointer

int *criarInt(void) {
    int local = 42;
    return &local;  // ERRO: retorna endereço de variável local
                    // local é destruída ao sair da função!
}
// p aponta para memória inválida — comportamento indefinido

Ponteiro para struct — operador seta (->)

Quando você tem um ponteiro para struct, precisa desreferenciar antes de acessar o campo. O operador -> faz as duas coisas de uma vez:

Sem -> (verboso)Com -> (equivalente, mais limpo)
(*r).nomer->nome
(*r).velocidade = 150r->velocidade = 150
(*r).posicao += 1.0fr->posicao += 1.0f
// SEM seta: verbose e fácil de errar com precedência
void configSemSeta(Robo *r) {
    (*r).velocidade = 150;    // parênteses obrigatórios! *r.vel seria erro
    (*r).posicao    = 0.0f;
}

// COM seta: equivalente, muito mais legível
void config(Robo *r) {
    r->velocidade = 150;      // r->campo == (*r).campo
    r->posicao    = 0.0f;
}

// Lendo e modificando — ponteiro permite alterar o original
void imprimirRobo(const Robo *r) {   // const: garante que não vai modificar
    printf("Nome: %s\n", r->nome);
    printf("Vel:  %d\n", r->velocidade);
}

Robo senna;
senna.velocidade = 200;
config(&senna);         // passa endereço — senna.velocidade vira 150
imprimirRobo(&senna);  // passa endereço — const: lê sem risco de modificar
Precedência de *: *r.campo não compila como esperado — o compilador lê como *(r.campo), que tenta desreferenciar o campo (erro). Sempre use (*r).campo ou prefira r->campo.
Passe structs por ponteiro (&), não por valor. Copiar uma struct grande na pilha a cada chamada é lento. Ponteiro copia apenas 4 ou 8 bytes (o endereço). Use const Robo *r quando a função só lê — comunica intenção e impede modificações acidentais.

malloc, calloc e free

#include <stdlib.h>

// malloc: aloca N bytes (conteúdo indefinido)
int *arr = (int*)malloc(10 * sizeof(int));

if (arr == NULL) {
    fprintf(stderr, "Erro: sem memória\n");
    return -1;
}

arr[0] = 42;
arr[9] = 99;

free(arr);    // SEMPRE libere a memória alocada!
arr = NULL;  // boa prática: evita dangling pointer

// calloc: aloca e inicializa com zero
float *buf = (float*)calloc(100, sizeof(float));

// realloc: redimensiona
arr = (int*)realloc(arr, 20 * sizeof(int));
Memory leak: Não chamar free para memória alocada = memory leak. Em programas longos, isso esgota a RAM. Em sistemas embarcados com pouca RAM, um único leak pode travar o sistema.

Arquivos — FILE*

#include <stdio.h>

FILE *fp = fopen("dados.txt", "w");  // abre para escrita

if (fp == NULL) {
    perror("Erro ao abrir arquivo");
    return -1;
}

fprintf(fp, "Posicao: %.2f\n", 4000.5f);
fprintf(fp, "Velocidade: %d\n", 150);

fclose(fp);  // SEMPRE feche o arquivo!

Modos de abertura

ModoSignificado
"r"Leitura (arquivo deve existir)
"w"Escrita (cria ou sobrescreve)
"a"Append (acrescenta ao final)
"r+"Leitura e escrita
"rb"Leitura binária
