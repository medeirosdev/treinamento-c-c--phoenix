Classes e Objetos

Em progresso

C++ vs C: o que muda?

C++ é um superconjunto de C com adições fundamentais: classes, herança, templates, sobrecarga de operadores, exceções, e muito mais. Todo código C válido compila como C++ (com raras exceções).

C++ foi criado por Bjarne Stroustrup na Bell Labs nos anos 1980 como "C com Classes". Hoje é a linguagem padrão de sistemas embarcados, games, trading de alta frequência e sistemas operacionais de alto desempenho.

Encapsulamento: a ideia central

Uma classe agrupa dados (atributos) e comportamentos (métodos) em uma unidade, controlando quem pode acessar o quê.

class Motor {
private:                    // só a classe acessa
    int velocidade;
    bool ligado;

public:                     // qualquer código pode usar
    Motor() : velocidade(0), ligado(false) {}  // construtor

    void ligar(int vel) {
        ligado = true;
        velocidade = vel;
    }

    void parar() {
        ligado = false;
        velocidade = 0;
    }

    int  getVelocidade() const { return velocidade; }
    bool estaLigado()    const { return ligado; }
};

Usando a classe

Motor esquerdo;            // chama construtor
esquerdo.ligar(200);
printf("%d\n", esquerdo.getVelocidade());  // 200
// esquerdo.velocidade = 300;  ERRO: private!

Construtores e lista de inicialização

class PID {
private:
    float kP, kD;
    float erroAnterior;

public:
    // Lista de inicialização: inicializa ANTES do corpo do construtor
    PID(float p, float d) : kP(p), kD(d), erroAnterior(0.0f) {}

    float calcular(float erro) {
        float derivativo = erro - erroAnterior;
        erroAnterior = erro;
        return kP * erro + kD * derivativo;
    }
};

PID controlador(2.5f, 0.8f);
float correcao = controlador.calcular(-500.0f);

Membros static

Um membro static pertence à classe, não a cada instância. É compartilhado por todos os objetos da classe.

class Motor {
private:
    static int totalMotores;  // UMA cópia para TODOS os objetos
    int id;

public:
    Motor()  { id = ++totalMotores; }
    ~Motor() { --totalMotores; }

    static int getTotal() { return totalMotores; }  // static: sem ponteiro "this"
};

// Obrigatório: inicializar o membro static FORA da classe (no .cpp)
int Motor::totalMotores = 0;

Motor a, b, c;
printf("%d\n", Motor::getTotal());  // 3 — acesso via ClassName::
Métodos static não recebem ponteiro this e não podem acessar membros não-static. São ideais para contadores de instâncias, configurações globais da classe e funções fábrica.

Chamadas em cascata com return *this

Retornar *this por referência permite encadear chamadas de método numa única expressão — padrão method chaining.

class PID {
    float kP{1.0f}, kD{0.0f}, erro{0.0f};
public:
    PID& setKP(float p) { kP = p; return *this; }   // retorna o próprio objeto
    PID& setKD(float d) { kD = d; return *this; }
    PID& resetar()     { erro = 0; return *this; }
};

PID pid;
pid.setKP(2.5f).setKD(0.8f).resetar();  // legível e compacto

Herança e Polimorfismo

Em progresso

Herança: reutilizando comportamento

Uma classe derivada herda atributos e métodos da classe base, podendo adicionar ou modificar comportamentos.

class Robo {           // classe base
protected:
    int velocidade;
    int posicao;

public:
    Robo() : velocidade(0), posicao(4000) {}

    virtual void atualizar() {  // virtual = pode ser sobrescrito
        posicao += velocidade;
    }

    int getPosicao() const { return posicao; }
};

class SeguidorLinha : public Robo {  // herda de Robo
private:
    float kP;

public:
    SeguidorLinha(float p) : kP(p) {}   // chama Robo() implicitamente

    void atualizar() override {          // sobrescreve o método base
        float erro = posicao - 4000;     // protected: acessa da subclasse
        velocidade = -(int)(kP * erro);
        Robo::atualizar();                // chama a versão da base
    }
};

Polimorfismo com ponteiro de base

Robo* robo = new SeguidorLinha(2.5f);
robo->atualizar();  // chama SeguidorLinha::atualizar() — não Robo::atualizar!
                    // Isso é polimorfismo dinâmico (virtual dispatch)
delete robo;
Por que virtual? Sem virtual, robo->atualizar() chamaria sempre Robo::atualizar(), ignorando o tipo real do objeto. Com virtual, o C++ usa uma tabela de funções virtuais (vtable) para chamar a implementação correta em tempo de execução.

Classe abstrata e métodos puros

class Controlador {         // classe abstrata: não pode ser instanciada
public:
    virtual float calcular(float erro) = 0;  // método puro (= 0)
    virtual ~Controlador() {}               // destrutor virtual: sempre!
};

class ControladorP : public Controlador {
    float kP;
public:
    ControladorP(float p) : kP(p) {}
    float calcular(float erro) override { return kP * erro; }
};

class ControladorPD : public Controlador {
    float kP, kD, ant;
public:
    ControladorPD(float p, float d) : kP(p), kD(d), ant(0) {}
    float calcular(float erro) override {
        float r = kP * erro + kD * (erro - ant);
        ant = erro; return r;
    }
};
Sempre declare o destrutor como virtual em classes base. Sem isso, deletar um objeto derivado através de um ponteiro base não chama o destrutor correto → memory leak.

override e final

void atualizar() override {}  // erro se não existe virtual na base
void atualizar() final    {}  // nenhuma subclasse pode sobrescrever
class Fechado final : public Base {}  // ninguém pode herdar de Fechado

Tipos de herança: public, protected, private

O especificador de herança controla como os membros da base ficam visíveis na classe derivada:

Use sempre public para modelar relacionamento é-um (is-a): SeguidorLinha é um Robo. Com herança private ou protected, o objeto derivado não pode ser tratado como a base — o polimorfismo não funciona.

Como o dispatch virtual funciona: vtable

Toda classe com funções virtual recebe uma tabela de ponteiros de função (vtable). Cada objeto guarda um ponteiro oculto para a vtable de sua classe real:

// Internamente o compilador constrói algo equivalente a:
//  Robo::vtable          → { &Robo::atualizar }
//  SeguidorLinha::vtable → { &SeguidorLinha::atualizar }

Robo* r = new SeguidorLinha(2.5f);

// r->atualizar() executa em 3 passos em runtime:
// 1. Segue o ponteiro r → objeto SeguidorLinha na heap
// 2. Lê o ponteiro vtable do objeto → SeguidorLinha::vtable
// 3. Chama a função apontada → SeguidorLinha::atualizar!
r->atualizar();  // não Robo::atualizar — o tipo real decide
Overhead da vtable: cada objeto com métodos virtual ganha +1 ponteiro oculto (4 ou 8 bytes). Cada chamada virtual passa por duas indireções de ponteiro extras. Para robótica com centenas de objetos, este custo é desprezível.

dynamic_cast — downcast seguro em runtime

void processarRobo(Robo* r) {
    // dynamic_cast testa em runtime se r aponta para SeguidorLinha
    // retorna nullptr se NÃO for — nunca crashea como static_cast
    SeguidorLinha* s = dynamic_cast<SeguidorLinha*>(r);

    if (s != nullptr) {
        s->ajustarKP(3.0f);  // seguro: sabemos que é SeguidorLinha
    }
}
dynamic_cast exige que a classe base tenha pelo menos uma função virtual. Use com moderação: muitos downcasts são sinal de hierarquia de herança mal projetada — prefira polimorfismo puro com métodos virtuais.

Function templates — código genérico

Em vez de escrever a mesma função para cada tipo, um template gera o código automaticamente:

// Sem template: precisaria de maximo_int, maximo_float, maximo_double...
template<typename T>
T maximo(T a, T b) {
    return (a > b) ? a : b;
}

maximo(3, 7);          // T = int   → retorna 7
maximo(3.14f, 2.7f);  // T = float → retorna 3.14
maximo('a', 'z');     // T = char  → retorna 'z'

Class templates

template<typename T, int N>
class RingBuffer {
private:
    T    dados[N];
    int  cabeca = 0;
    int  tamanho = 0;

public:
    void push(T valor) {
        dados[cabeca % N] = valor;
        cabeca++;
        if (tamanho < N) tamanho++;
    }

    T media() {
        T soma = T();   // zero-initializes o tipo T
        for (int i = 0; i < tamanho; i++) soma += dados[i];
        return soma / tamanho;
    }
};

RingBuffer<float, 10> leituras;  // buffer de 10 floats
leituras.push(3800.0f);
leituras.push(4100.0f);
float m = leituras.media();  // 3950.0

Referências vs Ponteiros

Referências são apelidos para variáveis existentes. Diferente de ponteiros, não podem ser nulas e não podem ser reassociadas.

int x = 10;
int& ref = x;   // ref é um apelido para x

ref = 99;         // modifica x
printf("%d\n", x);  // 99

// Referência vs Ponteiro: quais as diferenças?
int* p = &x;  // ponteiro: pode ser NULL, pode mudar pra onde aponta
int& r = x;   // referência: sempre válida, sempre o mesmo objeto
PonteiroReferência
Pode ser NULLSimNão
ReassociávelSim (p = &outro)Não (sempre o original)
Sintaxe de uso*p ou p->Igual à variável original
Quando usarOpcional/nulável, arraysParâmetros, aliases locais

Passagem por referência const

// Ruim: copia o vector inteiro
float somaVetor(std::vector<float> v);

// Bom: passa referência, não copia; const garante que não modifica
float somaVetor(const std::vector<float>& v) {
    float soma = 0;
    for (float x : v) soma += x;
    return soma;
}
Regra prática: Se o parâmetro tem mais de 4 bytes e você não precisa modificá-lo, passe como const T&. Para tipos primitivos (int, float), passe por valor — a referência não compensa.

new e delete em C++

// Aloca um objeto na heap e chama o construtor
Motor* m = new Motor();
m->ligar(200);
delete m;  // chama destrutor e libera memória

// Array na heap
int* arr = new int[100];
arr[0] = 42;
delete[] arr;  // delete[] para arrays — não delete!
delete[] para arrays, delete para objetos únicos. Confundir os dois é comportamento indefinido.

RAII — Resource Acquisition Is Initialization

O princípio mais importante de C++ moderno: ligar a vida de um recurso à vida de um objeto. O construtor adquire, o destrutor libera.

class GerenciadorArquivo {
private:
    FILE* fp;

public:
    GerenciadorArquivo(const char* path) {
        fp = fopen(path, "r");
    }
    ~GerenciadorArquivo() {
        if (fp) fclose(fp);  // fechado automaticamente!
    }
    FILE* get() { return fp; }
};

void lerArquivo() {
    GerenciadorArquivo arq("dados.txt");
    // usa arq.get() para ler...
    if (erro) return;  // fclose chamado mesmo aqui!
}  // fclose chamado aqui também

Sobrecarga de operadores

C++ permite redefinir o comportamento de operadores para tipos definidos pelo usuário, tornando o código mais natural.

class Vetor2D {
public:
    float x, y;
    Vetor2D(float x, float y) : x(x), y(y) {}

    // operator+ como membro: *this é o operando esquerdo
    Vetor2D operator+(const Vetor2D& outro) const {
        return Vetor2D(x + outro.x, y + outro.y);
    }

    // operator== para comparação
    bool operator==(const Vetor2D& outro) const {
        return x == outro.x && y == outro.y;
    }

    // operator<< deve ser função global (friend): cout fica à esquerda
    friend std::ostream& operator<<(std::ostream& out, const Vetor2D& v) {
        return out << "(" << v.x << ", " << v.y << ")";
    }
};

Vetor2D a{1.0f, 2.0f}, b{3.0f, 4.0f};
Vetor2D c = a + b;            // chama operator+  → (4.0, 6.0)
std::cout << c << std::endl;  // chama operator<< → (4.0, 6.0)
Os operadores ., .*, :: e ?: não podem ser sobrecarregados. Sobrecarregar == não sobrecarrega != automaticamente — implemente ambos explicitamente.

Construtor de cópia

Chamado automaticamente ao passar objeto por valor, retornar por valor ou inicializar a partir de outro objeto.

class Buffer {
    int* dados;
    int  tam;
public:
    Buffer(int n) : tam(n), dados(new int[n]) {}  // aloca heap

    // Cópia padrão (rasa): PROBLEMA — dois objetos apontam para a mesma memória!
    // Buffer(const Buffer&) = default;  ← NÃO fazer quando há ponteiros

    // Cópia profunda: cada objeto tem sua própria cópia dos dados
    Buffer(const Buffer& outro) : tam(outro.tam), dados(new int[outro.tam]) {
        for (int i = 0; i < tam; i++) dados[i] = outro.dados[i];
    }

    ~Buffer() { delete[] dados; }
};

Buffer a(100);
Buffer b = a;  // chama construtor de cópia — deep copy
Regra dos Três: se você escrever um destrutor personalizado, provavelmente precisará também de um construtor de cópia e de um operator= personalizados. Os três trabalham em trio para garantir semântica correta com recursos alocados.

Smart pointers (C++11)

#include <memory>

// unique_ptr: propriedade única, deleta ao sair do escopo
std::unique_ptr<Motor> m = std::make_unique<Motor>();
m->ligar(200);
// delete m; ← não precisa! unique_ptr faz isso automaticamente

// shared_ptr: contagem de referências, deleta quando ninguém usa
std::shared_ptr<Motor> a = std::make_shared<Motor>();
std::shared_ptr<Motor> b = a;  // compartilhado
// deletado quando ambos a e b saem do escopo
Em C++ moderno, prefira smart pointers a new/delete raw. Eles implementam RAII automaticamente e eliminam a classe inteira de bugs de memory leak e double-free.

Move semantics — rvalue references (C++11)

Move transfere a propriedade dos recursos de um objeto sem copiar dados — O(1) em vez de O(n).

#include <utility>

std::vector<int> origem = {1, 2, 3, 4, 5};

// Cópia: aloca nova memória, copia todos os dados — O(n)
std::vector<int> copia = origem;

// Move: "rouba" o buffer interno de origem, que fica vazio — O(1)
std::vector<int> movido = std::move(origem);
// origem agora está em estado válido mas vazio (não use depois!)

// Construtor de movimento: parâmetro T&& (rvalue reference)
class Buffer {
    int* dados; int tam;
public:
    Buffer(Buffer&& outro) noexcept    // && = rvalue reference
        : dados(outro.dados), tam(outro.tam) {
        outro.dados = nullptr;  // invalida a origem — evita double-free
        outro.tam   = 0;
    }
};
Containers da STL (vector, string, map) já implementam move semantics. Ao retornar um vector de uma função, o compilador usa RVO ou move — nenhuma cópia extra ocorre. Use std::move explicitamente apenas quando necessário passar propriedade de objetos.

std::optional — valor que pode não existir (C++17)

#include <optional>
#include <array>

// Retorna o índice do sensor mais escuro, ou "nada" se nenhum estiver na linha
std::optional<int> sensorNaLinha(const std::array<int, 9>& leituras) {
    for (int i = 0; i < 9; i++) {
        if (leituras[i] < 1000) return i;  // encontrou linha
    }
    return std::nullopt;                   // "não encontrou"
}

auto idx = sensorNaLinha(leituras);

if (idx.has_value()) {
    printf("Sensor %d na linha\n", idx.value());
}
// Atalho com operador bool e *:
if (idx) printf("%d\n", *idx);  // * é UB se vazio — cheque antes!
std::optional é uma alternativa semântica clara a retornar -1, nullptr ou um flag separado para indicar "sem valor". O tipo da função já documenta a possibilidade de ausência.

Exemplo: PID moderno em C++

class PIDController {
private:
    const float kP, kD;
    float erroAnterior{0.0f};

public:
    constexpr PIDController(float p, float d) : kP{p}, kD{d} {}

    [[nodiscard]] float calcular(float erro) {
        auto derivativo = erro - erroAnterior;
        erroAnterior    = erro;
        return kP * erro + kD * derivativo;
    }

    void resetar() noexcept { erroAnterior = 0.0f; }
};

auto pid = PIDController{2.5f, 0.8f};
auto correcao = pid.calcular(-500.0f);
