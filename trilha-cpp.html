Classes e Objetos

Em progresso

C++ vs C: o que muda?

C++ é um superconjunto de C com adições fundamentais: classes, herança, templates, sobrecarga de operadores, exceções, e muito mais. Todo código C válido compila como C++ (com raras exceções).

C++ foi criado por Bjarne Stroustrup na Bell Labs nos anos 1980 como "C com Classes". Hoje é a linguagem padrão de sistemas embarcados, games, trading de alta frequência e sistemas operacionais de alto desempenho.

Encapsulamento: a ideia central

Uma classe agrupa dados (atributos) e comportamentos (métodos) em uma unidade, controlando quem pode acessar o quê.

class Motor {
private:                    // só a classe acessa
    int velocidade;
    bool ligado;

public:                     // qualquer código pode usar
    Motor() : velocidade(0), ligado(false) {}  // construtor

    void ligar(int vel) {
        ligado = true;
        velocidade = vel;
    }

    void parar() {
        ligado = false;
        velocidade = 0;
    }

    int  getVelocidade() const { return velocidade; }
    bool estaLigado()    const { return ligado; }
};

Usando a classe

Motor esquerdo;            // chama construtor
esquerdo.ligar(200);
printf("%d\n", esquerdo.getVelocidade());  // 200
// esquerdo.velocidade = 300;  ERRO: private!

Construtores e lista de inicialização

class PID {
private:
    float kP, kD;
    float erroAnterior;

public:
    // Lista de inicialização: inicializa ANTES do corpo do construtor
    PID(float p, float d) : kP(p), kD(d), erroAnterior(0.0f) {}

    float calcular(float erro) {
        float derivativo = erro - erroAnterior;
        erroAnterior = erro;
        return kP * erro + kD * derivativo;
    }
};

PID controlador(2.5f, 0.8f);
float correcao = controlador.calcular(-500.0f);

Destrutor

class Buffer {
private:
    int* dados;
    int  tamanho;

public:
    Buffer(int n) : tamanho(n) {
        dados = new int[n];  // aloca no construtor
    }

    ~Buffer() {            // destrutor: ~ antes do nome
        delete[] dados;    // libera automaticamente ao sair do escopo!
    }
};

{
    Buffer b(100);
    // usa b...
}  // ← ~Buffer() chamado aqui: sem memory leak!

Herança e Polimorfismo

Em progresso

Herança: reutilizando comportamento

Uma classe derivada herda atributos e métodos da classe base, podendo adicionar ou modificar comportamentos.

class Robo {           // classe base
protected:
    int velocidade;
    int posicao;

public:
    Robo() : velocidade(0), posicao(4000) {}

    virtual void atualizar() {  // virtual = pode ser sobrescrito
        posicao += velocidade;
    }

    int getPosicao() const { return posicao; }
};

class SeguidorLinha : public Robo {  // herda de Robo
private:
    float kP;

public:
    SeguidorLinha(float p) : kP(p) {}   // chama Robo() implicitamente

    void atualizar() override {          // sobrescreve o método base
        float erro = posicao - 4000;     // protected: acessa da subclasse
        velocidade = -(int)(kP * erro);
        Robo::atualizar();                // chama a versão da base
    }
};

Polimorfismo com ponteiro de base

Robo* robo = new SeguidorLinha(2.5f);
robo->atualizar();  // chama SeguidorLinha::atualizar() — não Robo::atualizar!
                    // Isso é polimorfismo dinâmico (virtual dispatch)
delete robo;
Por que virtual? Sem virtual, robo->atualizar() chamaria sempre Robo::atualizar(), ignorando o tipo real do objeto. Com virtual, o C++ usa uma tabela de funções virtuais (vtable) para chamar a implementação correta em tempo de execução.

Classe abstrata e métodos puros

class Controlador {         // classe abstrata: não pode ser instanciada
public:
    virtual float calcular(float erro) = 0;  // método puro (= 0)
    virtual ~Controlador() {}               // destrutor virtual: sempre!
};

class ControladorP : public Controlador {
    float kP;
public:
    ControladorP(float p) : kP(p) {}
    float calcular(float erro) override { return kP * erro; }
};

class ControladorPD : public Controlador {
    float kP, kD, ant;
public:
    ControladorPD(float p, float d) : kP(p), kD(d), ant(0) {}
    float calcular(float erro) override {
        float r = kP * erro + kD * (erro - ant);
        ant = erro; return r;
    }
};
Sempre declare o destrutor como virtual em classes base. Sem isso, deletar um objeto derivado através de um ponteiro base não chama o destrutor correto → memory leak.

override e final

void atualizar() override {}  // erro se não existe virtual na base
void atualizar() final    {}  // nenhuma subclasse pode sobrescrever
class Fechado final : public Base {}  // ninguém pode herdar de Fechado

Function templates — código genérico

Em vez de escrever a mesma função para cada tipo, um template gera o código automaticamente:

// Sem template: precisaria de maximo_int, maximo_float, maximo_double...
template<typename T>
T maximo(T a, T b) {
    return (a > b) ? a : b;
}

maximo(3, 7);          // T = int   → retorna 7
maximo(3.14f, 2.7f);  // T = float → retorna 3.14
maximo('a', 'z');     // T = char  → retorna 'z'

Class templates

template<typename T, int N>
class RingBuffer {
private:
    T    dados[N];
    int  cabeca = 0;
    int  tamanho = 0;

public:
    void push(T valor) {
        dados[cabeca % N] = valor;
        cabeca++;
        if (tamanho < N) tamanho++;
    }

    T media() {
        T soma = T();   // zero-initializes o tipo T
        for (int i = 0; i < tamanho; i++) soma += dados[i];
        return soma / tamanho;
    }
};

RingBuffer<float, 10> leituras;  // buffer de 10 floats
leituras.push(3800.0f);
leituras.push(4100.0f);
float m = leituras.media();  // 3950.0

Referências, Memória e RAII

Em progresso

Referências vs Ponteiros

Referências são apelidos para variáveis existentes. Diferente de ponteiros, não podem ser nulas e não podem ser reassociadas.

int x = 10;
int& ref = x;   // ref é um apelido para x

ref = 99;         // modifica x
printf("%d\n", x);  // 99

// Referência vs Ponteiro: quais as diferenças?
int* p = &x;  // ponteiro: pode ser NULL, pode mudar pra onde aponta
int& r = x;   // referência: sempre válida, sempre o mesmo objeto
PonteiroReferência
Pode ser NULLSimNão
ReassociávelSim (p = &outro)Não (sempre o original)
Sintaxe de uso*p ou p->Igual à variável original
Quando usarOpcional/nulável, arraysParâmetros, aliases locais

Passagem por referência const

// Ruim: copia o vector inteiro
float somaVetor(std::vector<float> v);

// Bom: passa referência, não copia; const garante que não modifica
float somaVetor(const std::vector<float>& v) {
    float soma = 0;
    for (float x : v) soma += x;
    return soma;
}
Regra prática: Se o parâmetro tem mais de 4 bytes e você não precisa modificá-lo, passe como const T&. Para tipos primitivos (int, float), passe por valor — a referência não compensa.

new e delete em C++

// Aloca um objeto na heap e chama o construtor
Motor* m = new Motor();
m->ligar(200);
delete m;  // chama destrutor e libera memória

// Array na heap
int* arr = new int[100];
arr[0] = 42;
delete[] arr;  // delete[] para arrays — não delete!
delete[] para arrays, delete para objetos únicos. Confundir os dois é comportamento indefinido.

RAII — Resource Acquisition Is Initialization

O princípio mais importante de C++ moderno: ligar a vida de um recurso à vida de um objeto. O construtor adquire, o destrutor libera.

class GerenciadorArquivo {
private:
    FILE* fp;

public:
    GerenciadorArquivo(const char* path) {
        fp = fopen(path, "r");
    }
    ~GerenciadorArquivo() {
        if (fp) fclose(fp);  // fechado automaticamente!
    }
    FILE* get() { return fp; }
};

void lerArquivo() {
    GerenciadorArquivo arq("dados.txt");
    // usa arq.get() para ler...
    if (erro) return;  // fclose chamado mesmo aqui!
}  // fclose chamado aqui também

Smart pointers (C++11)

#include <memory>

// unique_ptr: propriedade única, deleta ao sair do escopo
std::unique_ptr<Motor> m = std::make_unique<Motor>();
m->ligar(200);
// delete m; ← não precisa! unique_ptr faz isso automaticamente

// shared_ptr: contagem de referências, deleta quando ninguém usa
std::shared_ptr<Motor> a = std::make_shared<Motor>();
std::shared_ptr<Motor> b = a;  // compartilhado
// deletado quando ambos a e b saem do escopo
Em C++ moderno, prefira smart pointers a new/delete raw. Eles implementam RAII automaticamente e eliminam a classe inteira de bugs de memory leak e double-free.

C++ Moderno (C++11/14/17)

Em progresso

auto — dedução de tipo

auto x = 42;          // int
auto pi = 3.14;       // double
auto nome = std::string("Senna");

// Especialmente útil com tipos longos
std::vector<std::pair<int, std::string>> lista;
auto it = lista.begin();  // muito melhor que escrever o tipo completo

// auto& para referência, const auto& para const ref
for (const auto& item : lista) { /* ... */ }

Range-based for loop

std::vector<int> v = {10, 20, 30};

for (int x : v) printf("%d\n", x);       // cópia
for (int& x : v) x *= 2;              // referência: modifica v
for (const int& x : v) printf("%d ", x); // const ref: eficiente + readonly

nullptr — ponteiro nulo tipado

// C antigo: NULL é simplesmente 0, pode causar ambiguidade
void f(int n);
void f(char* s);
f(NULL);    // ambíguo: f(0) ou f(nullptr)?
f(nullptr); // claro: chama f(char*)

int* p = nullptr;  // prefira sempre nullptr a NULL

constexpr — computação em tempo de compilação

constexpr int NUM_SENSORES = 9;
constexpr float PI = 3.14159265f;
constexpr int quadrado(int x) { return x * x; }

int arr[quadrado(4)];   // 16 — tamanho calculado em compile-time

// constexpr é melhor que #define: tem tipo, escopo, pode ser função

Lambda functions

#include <algorithm>

std::vector<int> v = {5, 1, 3, 2, 4};

// Lambda: [captura](params) { corpo }
std::sort(v.begin(), v.end(),
    [](int a, int b) { return a < b; });  // ordena crescente

// Lambda com captura de variável
int limite = 3;
auto ehMenorQueLimite = [limite](int x) { return x < limite; };
// [=] captura tudo por valor, [&] por referência

// Lambda como variável
auto somar = [](float a, float b) { return a + b; };
float r = somar(1.5f, 2.5f);  // 4.0

Inicialização uniforme com {}

int    a{42};            // equivale a int a = 42
float  f{3.14f};
Motor  m{};              // construtor padrão

std::vector<int> v{1, 2, 3};  // inicializa com os três valores

// Vantagem: detecta narrowing conversions em compile time
int x{3.14};  // ERRO de compilação: 3.14 não cabe em int sem truncar
int y = 3.14; // Apenas warning — muito mais silencioso

enum class — enumerações com escopo

// C antigo: enum sem escopo, nomes podem colidir
enum Status { OK, ERRO };        // OK e ERRO no escopo global

// C++11: enum class tem escopo próprio
enum class ModoRobo { Parado, Seguindo, Retorno };
enum class StatusSensor { OK, Falha, Saturado };  // OK não conflita!

ModoRobo modo = ModoRobo::Seguindo;
if (modo == ModoRobo::Seguindo) { /* ... */ }
// if (modo == 1)  ERRO: enum class não converte implicitamente para int

Structured bindings (C++17)

std::pair<int, std::string> resultado = {42, "ok"};
auto [codigo, mensagem] = resultado;  // desestrutura o par

std::map<std::string, int> mapa = {{"a", 1}, {"b", 2}};
for (const auto& [chave, valor] : mapa) {
    printf("%s=%d\n", chave.c_str(), valor);
}

Exemplo: PID moderno em C++

class PIDController {
private:
    const float kP, kD;
    float erroAnterior{0.0f};

public:
    constexpr PIDController(float p, float d) : kP{p}, kD{d} {}

    [[nodiscard]] float calcular(float erro) {
        auto derivativo = erro - erroAnterior;
        erroAnterior    = erro;
        return kP * erro + kD * derivativo;
    }

    void resetar() noexcept { erroAnterior = 0.0f; }
};

auto pid = PIDController{2.5f, 0.8f};
auto correcao = pid.calcular(-500.0f);
